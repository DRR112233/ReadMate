# 记忆系统审查与加固记录

更新时间：2026-04-17

## 目标

让 ReadMate 的记忆系统从“可用”升级到“稳定可信”，减少以下问题：

- 误记忆（记错、记偏）
- 伪连续（看似记得，实际跨场景串味）
- 记忆丢失（备份恢复后失忆）
- 事件噪音（无意义 interaction 刷屏）

## 审查结论（按优先级）

### P0 - 必修

1. **备份/恢复缺失记忆数据**
   - 问题：`app_memory_store_v1` 未纳入备份白名单，用户恢复后会失忆。
   - 影响：高，直接破坏“像人一样记得住”的核心体验。
   - 状态：已修复（本次）。

### P1 - 高优先

2. **interaction 事件过于频繁**
   - 问题：每次 `rememberConversationTurn` 都写一条 interaction，容易淹没关键事件。
   - 影响：中高，事件流可读性与召回质量下降。
   - 状态：已修复（本次，按 channel 限频聚合）。

3. **事实召回跨场景串味**
   - 问题：事实记忆默认全局 TopK，缺少 channel/book 相关性优先级。
   - 影响：中，回复可能“像记得但不合时宜”。
   - 状态：已修复（本次，场景化召回优先级已上线）。

### P2 - 持续优化

4. 冲突偏好消解（例如“以前喜欢A，现在不喜欢A”）
5. 事实置信度衰减与冷启动稳态策略
6. 记忆提取精度提升（规则 + LLM 二次判定）
7. 记忆反馈闭环（用户标记“记对了/记错了”）

## 本次修复内容

### 1) 备份恢复纳入记忆数据

- 将 `app_memory_store_v1` 纳入 `backupLocalStorageKeys`。
- 保障导出备份 -> 恢复备份后，长期记忆、事件、会话摘要可保留。

### 2) interaction 事件去噪

- 为 interaction 写入增加了 **按 channel 限频**：
  - 同一 channel 在冷却窗口内不重复写 interaction 事件。
- 事件打上 channel tag，便于后续继续做分层统计和可视化筛选。
- 减少噪音，保留“有意义的互动轨迹”。

### 本次新增思路（已记录，待落地）

- **场景化检索优先级**：召回顺序调整为 `当前书相关` > `当前 channel 相关` > `全局置顶` > `全局高分`（已落地）。
- **冲突偏好合并**：对“喜欢/不喜欢同一对象”建立 key 级别覆盖策略（已落地基础版）。
- **记忆衰减策略**：对长期未命中的未置顶事实进行分值衰减（已落地）。
- **记忆质量闭环**：在记忆管理页增加“记对了/记错了”反馈，直接影响记忆分值和置信度（已落地基础版）。
- **避免重复犯错（踩雷清单）**：将“记错了”的事实写入独立的“踩雷清单”，后续注入到提示词的「避免踩雷」段落，并在召回时屏蔽同文本误记忆（已落地）。

## 下一步建议（实现顺序）

1. **事实召回分层**（book/channel 强相关优先）
2. **偏好冲突合并器**
3. **记忆有效期与衰减**
4. **记忆质量评估指标面板**（误记忆率、命中率、用户修正率）

## 验收清单

- [x] 恢复备份后记忆不丢失
- [x] interaction 事件数量显著下降
- [x] 同场景命中优先于全局记忆
- [x] 用户纠正语句可覆盖旧偏好（偏好语句基础覆盖）
- [x] 记忆可被用户反馈纠偏（记对/记错）
